# Quilo에 기여하기

작은 오타 수정, 실패 재현, 테스트, 설계 의견도 모두 기여입니다. 처음부터 구현을
맡기보다 먼저 이슈에서 문제와 기대 결과를 맞추면 중복 작업을 줄일 수 있습니다.

## 시작하기

1. `CODE_OF_CONDUCT.md`와 `SECURITY.md`를 읽습니다.
2. 기존 이슈를 검색하고, 없으면 재현 방법·기대 결과·실제 결과를 적어 새 이슈를
   만듭니다. 보안 취약점은 공개 이슈에 쓰지 않습니다.
3. 저장소를 fork하고 `main`에서 짧은 작업 브랜치를 만듭니다.
4. `npm run harness:doctor`와 아래 검증 명령을 실행합니다.
5. 한 PR에는 가능한 한 하나의 문제와 최소 변경만 담습니다.

```bash
npm install
npm run release:oss-check
npm run verify:quick
npm run verify:core
```

프론트엔드나 실제 생성 경로를 바꾸면 관련 Playwright 경로와 생성 산출물을 직접
확인해 주세요. 빌드 통과만으로 외부 API, 인증, 저장, OCR, HWPX/DOCX 출력이
실제로 동작한다고 주장하지 않습니다.

## Pull request에 포함할 내용

- 문제와 사용자 영향
- 변경한 이유와 대안
- 실행한 검증과 아직 검증하지 못한 항목
- UI 변경 전후 이미지(해당하는 경우)
- 새 의존성의 업스트림 URL, 버전, SPDX 라이선스
- 데이터·템플릿·미디어를 추가할 권리가 있다는 확인

커밋에는 Developer Certificate of Origin 방식의 sign-off를 붙여 주세요.

```bash
git commit -s -m "fix: describe the change"
```

sign-off는 해당 기여를 제출할 권리가 있고 프로젝트 라이선스로 배포하는 데
동의한다는 확인입니다.

## 라이선스와 데이터

별도 고지가 없는 기여는 프로젝트와 같은 `AGPL-3.0-or-later`로 제공됩니다.
상업적 이용 금지, 교육용 한정, 출처 표시만 허용 같은 커스텀 조건의 코드를
복사하지 마세요. 사용자 파일, 실제 보고서, 학교 양식, API 키, 쿠키, 운영 로그도
커밋하지 않습니다.

AI 도구의 도움을 받은 기여는 사람이 전체 diff와 테스트 결과를 검토하고, 출처가
불명확한 코드를 포함하지 않았음을 PR에 밝혀 주세요.

## 리뷰 원칙

기준으로 리뷰합니다. 요청된 수정이 오래 걸리면 작업을 더 작은 PR로 나눌 수
있습니다. 행동 규범을 지키는 한 질문과 반대 의견은 환영합니다.
