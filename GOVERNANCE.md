# Quilo 거버넌스

Quilo는 현재 창립 메인테이너가 최종 merge와 release 권한을 가진
maintainer-led 프로젝트입니다. 설계와 우선순위는 가능한 한 공개 Issue와 PR에서
논의합니다.

## 의사결정

- 되돌리기 쉬운 작은 변경은 PR 리뷰와 검증 후 메인테이너가 결정합니다.
- API, 데이터 모델, 라이선스, 개인정보, 호환성을 바꾸는 결정은 이슈에서 의견을
  모으고 결정 근거를 기록합니다.
- 합의가 어렵다면 사용자 안전, 데이터 무결성, 오픈소스 라이선스 적합성,
  유지보수 가능성을 우선해 메인테이너가 결정합니다.

## 권한 확대

지속적으로 품질 높은 리뷰와 기여를 하고 행동 규범을 지킨 참여자는 triage 또는
maintainer 권한을 제안받을 수 있습니다. 권한은 최소 범위로 부여하며 이해충돌,
장기 부재, 보안 문제 또는 행동 규범 위반 시 조정할 수 있습니다.

## 릴리스

릴리스에는 변경 요약, 실행한 검증 수준, 데이터베이스 migration, 알려진 제한,
라이선스·제3자 자산 변경을 기록합니다. 운영 배포 권한과 오픈소스 merge 권한은
서로 별개입니다.
