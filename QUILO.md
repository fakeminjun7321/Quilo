## 1. Quilo가 하는 일

Quilo는 **영재학교(영재고) 학생의 학업 전반을 돕는 AI**입니다. 실험·수행평가·독서록·내신 준비처럼 "결과물을 학교 양식으로 제출해야 하는" 반복 작업을, 네가 실제로 한 것만 재료로 삼아 **바로 낼 수 있는 형태(.hwpx / .docx)로 완성**해 줍니다.

범용 AI는 대화창 안의 답까지가 끝이라, 그걸 다시 한글 양식에 옮기고 표를 그리고 수식을 캡처하는 "마지막 1km"는 늘 학생 몫으로 남습니다. Quilo는 그 마지막 1km — **학교가 그대로 받아주는 제출용 문서** — 를 대신 끝냅니다.

대표 기능은 **실험/수행평가 보고서 생성**입니다. 실험 자료(매뉴얼 PDF, 측정 데이터 엑셀/CSV, 실험 사진, PASCO Capstone `.cap` 파일 등)를 올리면, AI가 그 자료를 읽고 학교 양식에 맞춰 **표·그래프·계산·분석·결론이 들어간 보고서 초안**을 만듭니다. (독서록·시험대비·양식 만들기 등 다른 학업 작업은 2번·8번 참고.)

핵심 원칙:
- 출력은 바로 제출/편집 가능한 **한글(.hwpx) 또는 워드(.docx)** 파일
- AI가 **데이터를 지어내지 않음** — 업로드한 자료 안의 값만 사용 (데이터 무결성 원칙, 아래 7번)
- 표·그래프는 실제 측정값으로 계산/렌더링
- 수식은 한글 수식 객체로 변환되어 들어감

**Quilo는 "네가 한 걸 대신 제출해 주는 곳"이 아니라 "네가 한 걸 제출할 수 있게 완성해 주는 도구"**입니다. 재료(데이터·필기·사진)는 네가 올려야 하고, 받은 초안은 직접 검토·수정해서 제출합니다.

## 2. 핵심 기능: 보고서 생성

메인 화면에서 보고서 종류를 고르고 → 자료를 올리고 → 정보를 입력하고 → 생성 버튼을 누르면, 진행 로그가 실시간으로 표시되며 완성된 파일을 다운로드합니다.

### 보고서 종류

| 종류 | 코드 | 무엇을 올리나 | 결과물 |
|---|---|---|---|
| **화학 사전보고서** | `chem-pre` | 실험 매뉴얼 PDF | 실험목표·이론적 배경·기구/시약·실험과정이 담긴 사전보고서. 시약 물성은 웹검색으로 보강, 필요한 그림은 placeholder로 표시 |
| **화학 결과보고서** | `chem-result` | 사전보고서(PDF/docx) + 측정 데이터(엑셀/CSV/사진) + 실험 사진 | 사전보고서 뒤에 붙는 결과·데이터처리·표·그래프·분석·결론 |
| **물리 결과보고서** | `phys-result` | PASCO Capstone `.cap` / 엑셀·CSV·txt / 매뉴얼 PDF / 사진 (하나 이상 필수) | Part별 표·그래프·분석·결론(오차분석 포함). `.cap` 워크북 구조에 맞춰 실험 파트를 나눔 |
| **자유 보고서** | `free` | 임의 주제 — 작성 지시 + 평가기준 + 참고자료 | 주제·양식 제한 없는 범용 보고서(표·수식·그래프·사진 지원) |
| 물리 수행평가 *(베타)* | `phys-inquiry` | 탐구 주제 + 필기노트 PDF | 「탐구 및 사고 과정 성찰 보고서」(오개념→해결→성찰) |
| 수학 수행평가 *(베타)* | `math-inquiry` | 탐구 주제 + 필기노트 | 「수학Ⅲ 급수 탐구보고서」(Ⅰ~Ⅴ 양식, 표·차트·수식) |
| 독서록 *(베타)* | `reading-log` | 도서 정보(제목·저자 등) | 학교 「독서활동 기록지」 HWPX 양식을 채운 `.hwpx` |
| 독서록 대량생성 *(베타)* | `reading-log-bulk` | 책 목록 엑셀(XLSX/CSV) | 책마다 독서록 HWPX를 만들어 **ZIP**으로 묶음(기간 자동 분배) |
| 문제집 메이커 *(베타)* | `problem-set` | 교재 문제 PDF/사진 | 문제지 + 해설지 3종 PDF(**ZIP**) |
| 양식 메이커 *(베타)* | `form-maker` | 양식 설명(텍스트) 또는 종이 문서 사진 | 말한 대로 만든 한글 양식, 또는 사진→문서 복원(.docx/.hwpx) |
| 영어 시험대비 3종 *(베타)* | `eng-exam-prep` | 영어 지문/학습지 파일 | 모의고사·개념정리·빈칸 학습지 3종 PDF(**ZIP**) |
| 국어 문학 내신·모의고사 *(베타)* | `korean-lit-exam` | 학습지(판서) + 문제은행 | 시험지·답안지·정답해설지 3종 PDF(**ZIP**) |
| Capstone `.cap` 번역본 *(베타)* | `cap-translate` | PASCO Capstone `.cap` | 화면 텍스트를 한국어로 번역한 `.cap`(**ZIP**) |
| 물리 모의고사 *(베타)* | `phys-mock-exam` | 기출 + 교과서 단원 파일 | 같은 스타일의 새 시험지 PDF + HWPX(**ZIP**) |

> 화학 사전·화학 결과·물리 결과·자유(`chem-pre`/`chem-result`/`phys-result`/`free`) 4종은 **누구나 사용 가능(크레딧 차감)**. 나머지 *(베타)* 표시 타입은 **관리자/지정 테스터에게만 보이고 무료**입니다.

### 공통 입력 옵션 (모든 보고서)

- **날짜 / 학번 / 이름** — 표지에 들어감 (물리 결과보고서는 학번 필수)
- **출력 형식** — `.docx`(워드) 또는 `.hwpx`(한글)
- **글꼴** — 나눔명조/나눔고딕/바탕 등
- **AI 참고 메모 (`userNotes`)** — ⭐ 중요 개념, 아래 6번 참고
- **내 글 스타일 (`styleRefs`)** — 사용자가 올린 본인 글 샘플의 **문체만** 흉내 (내용·데이터는 누출 안 함)
- **AI 이미지 생성** *(화학 사전보고서, 베타·옵트인)* — 이론 설명용 개념도·삽화를 AI가 생성해 삽입. 실험 사진/측정 데이터는 절대 생성 안 함. 장당 1크레딧 추가

## 3. 모델과 크레딧

보고서를 만들 때 AI 모델을 고를 수 있고, 모델에 따라 크레딧이 차감됩니다.

| 모델 | 크레딧 | 비고 |
|---|---|---|
| **Claude Opus 4.8** | 4 | 기본값, 가장 똑똑함 |
| Claude Sonnet 5 | 2 | 빠르고 저렴 |
| GPT-5.5 | 4 | 화학/물리·자유·시험대비 등 대부분 타입에서 선택 가능 |
| GPT-5.4 | 1 | 〃 |
| GPT-5.4-mini | 0 (무료) | 〃 |

- 크레딧은 회원 계정에 쌓여 있고, 보고서 1건 생성마다 모델 크레딧만큼 차감됩니다.
- **베타 표시 타입(`phys-inquiry`, `math-inquiry`, 독서록, 문제집, 양식, 영어/국어 시험대비, `.cap` 번역, 물리 모의고사)은 무료라 크레딧을 쓰지 않습니다.**
- **AI 이미지 생성**을 켜면 삽입 이미지 1장당 1크레딧이 추가됩니다.
- 로그인 + 약관 동의 + 이메일 인증 + 관리자 승인이 필요합니다(관리자는 예외).

## 4. 출력 형식: .docx와 .hwpx

- **`.docx`** — Microsoft Word / 한글에서도 열림. 안정적.
- **`.hwpx`** — 한글(HWP) 전용 포맷. 학교가 한글 양식을 요구할 때 사용. 학교 표준 양식 템플릿을 기준으로 제목 박스·표·차트·사진·수식이 실제로 박혀 들어갑니다.
- 수식은 `{{EQ:...}}` 같은 마커 형태로 만든 뒤 **실제 한글 수식 객체로 변환**됩니다. (raw 마커가 그대로 보이면 버그)

## 5. 보고서 생성 흐름 (사용자가 보는 단계)

1. 보고서 종류 선택
2. 자료 업로드 (매뉴얼/데이터/사진/`.cap` 등)
3. 정보 입력 (날짜·학번·이름·형식·글꼴, 선택적으로 메모/스타일)
4. **생성** 버튼 → 진행 화면: `업로드 확인 → AI 분석 → 문서 생성 → 다운로드 준비`
5. 완성 파일 다운로드 (로그인 시 24시간 동안 "내 파일함"에도 보관)

업로드 파일은 단일 파일 최대 64MB까지 허용. 아주 큰 스캔 PDF는 페이지를 나눠 올리는 게 안정적입니다.

## 6. ⭐ "AI 참고 메모"가 뭔지 (AI가 꼭 이해해야 할 개념)

Quilo의 보고서 폼에는 **"AI 참고 메모(`userNotes`)"** 칸이 있습니다. 사용자가 다른 AI(당신)에게 도움을 청하는 가장 흔한 이유가 이 메모 작성입니다.

- **메모 = 데이터만 봐서는 알 수 없는 맥락**을 AI에게 알려주는 글입니다. 예:
  - 제외한 측정 시행과 그 이유
  - 실험 중 관찰한 특이사항 / 장치 세팅 문제
  - 실제로 겪은 오차 원인과 해결 과정
  - 특별히 강조해서 분석해 줬으면 하는 부분
- 메모는 **보조 맥락**이지 데이터를 덮어쓰지 않습니다. 같은 사실은 보고서에 1~2번만 반영됩니다.
- **메모에 없는 오차 원인·제외 기준·개선 효과·수치 변화를 지어내면 안 됩니다.**

👉 **사용자가 "Quilo에 넣을 메모를 써 달라"고 하면**: 사용자 상황을 듣고, 부족하면 2~3개만 질문한 뒤, **불릿 형태의 메모 초안**을 만들어 주세요. 단, 사용자가 실제로 겪지 않은 일이나 데이터에 없는 수치는 절대 지어내지 마세요.

(사이트 안에도 보고서별로 "AI에게 물어볼 때 복사해 쓰는 기본 프롬프트"가 적혀 있습니다.)

## 7. 데이터 무결성 — Quilo의 핵심 원칙

이 도구가 지키는 절대 규칙입니다. 사용자를 도울 때 당신도 이 선을 지켜 주세요.

- 업로드한 원본 데이터를 **조용히 평균내거나 임의로 바꾸지 않음**
- 자료에 **없는 측정값·수치를 만들어내지 않음**
- 메모에 없는 오차 원인·제외 기준·해결 결과를 **지어내지 않음**
- 사용자가 직접 정리한 엑셀/CSV가 `.cap` 원자료보다 우선
- 표·그래프는 **실제 업로드 데이터**로만 계산/렌더링

## 8. 부가 기능 (보고서 외)

Quilo는 보고서 생성 외에도 학습/문서 도구를 제공합니다.

- **도구 모음** (`/tools`) — PDF 합치기·나누기·회전·압축·자르기·페이지번호·워터마크·추출, 이미지 변환, 그래프 그리기, 회귀분석, 글자수 세기 등 (브라우저 안에서 처리)
- **PDF 통번역** (`/translate.html`, 베타) — 그림·레이아웃은 그대로 두고 텍스트만 한국어로 번역 (DeepL 스타일)
- **창작 스튜디오** (`/studio.html`, `/create.html`) — 대화형 "바이브 코딩" 스튜디오. 미리보기·체크포인트로 웹 아티팩트 제작
- **바이브 코딩 생성기 / 고급 물리 문제 스튜디오** (`/vibe-coding.html`, `/physics-studio.html`) — 아이디어 한 줄로 프로젝트 설계, 또는 주제·난이도로 심화 물리 문제·풀이·힌트 생성
- **수행평가 도움 허브** (`/exam-prep.html`, 베타) — 백준식 코딩 테스트(브라우저 안 Pyodide 채점, 소크라테스식 튜터) + 물리 수행평가
- **공부(상대론)** (`/study.html`, 베타) — 상대성이론 민코프스키 평면 학습 도식 생성·렌더링
- **파일 챗봇** (`/filechat.html`, 베타) — 파일을 올리고 Claude와 대화
- **AI 채팅 도우미** — 보고서 폼에 내장. "AI 참고 메모"를 어떻게 쓸지 등 사용법을 돕는 라이트 챗봇
- **커뮤니티 + 랩** (`/community.html`) — 건의/기능요청 게시판 + "랩"(서비스에 쓰인 기술을 GitHub식으로 열람·다운로드)
- **백그라운드 실행** — 탭을 닫아도 서버가 보고서를 끝까지 만들어 "내 파일함"·이메일로 전달(구독자·관리자 한정)
- **예시 모음** (`/examples.html`) — 보고서 샘플과 AI 메모 작성 가이드
- **이용 가이드** (`/guide.html`), **패치노트** (`/changelog.html`)

3. **보고서 내용 자체를 당신이 통째로 써 줄 필요는 없습니다** — 그건 Quilo가 자료를 받아 생성합니다. 당신은 "무엇을 올려야 하나", "메모를 어떻게 쓰나", "왜 이렇게 나왔나"를 돕는 역할입니다.
