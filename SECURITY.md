# Security Policy

## 보안 문제 제보

보안 취약점이나 개인정보 노출 가능성을 발견했다면 공개 이슈에 상세 재현 정보, 계정 정보, 업로드 파일 또는 토큰을 게시하지 마세요.

1. GitHub의 비공개 보안 제보 기능을 사용할 수 있으면 해당 채널을 우선 사용합니다.
2. 사용할 수 없으면 [Quilo](https://quilolab.com)의 공식 문의 채널로 최소한의 정보만 전달합니다.
3. 실제 사용자 데이터 없이 재현 가능한 범위에서 영향과 발생 위치를 설명합니다.

이 공개 저장소에는 운영 소스나 자격 증명이 포함되지 않습니다. 실제 서비스의 보안 제보는 공개 문서 수정 요청보다 우선해 처리합니다.
