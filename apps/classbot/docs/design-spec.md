# Quilo 관리자 화면 디자인 명세

기준 시안: `admin-dashboard-concept.png` (1536×1024)

## 화면 구조

- 240px 좌측 내비게이션, 유동형 중앙 작업 영역, 356px 우측 편집 패널을 사용한다.
- 첫 화면은 마케팅 페이지가 아니라 `오늘의 학급 운영` 작업 화면이다.
- 시간표는 카드 그리드가 아닌 행 중심 테이블로 표현한다.
- 수행평가와 알림 기록은 한 단계 낮은 목록형 패널로 배치한다.
- 모바일에서는 좌측 내비게이션을 하단 탭으로, 우측 편집 패널을 전체 화면 시트로 전환한다.

## 디자인 토큰

- 배경: `#ffffff`
- 내비게이션: `#f7f9fc`
- 본문 텍스트: `#0b1b3a`
- 보조 텍스트: `#5d687c`
- 테두리: `#dbe2ed`
- 포인트: `#1457e6`
- 포인트 연한 배경: `#edf3ff`
- 위험/마감: `#ee4d3d`
- 성공: `#079455`
- 반경: 8px 입력, 10px 버튼, 14px 목록 패널
- 그림자: 편집 패널의 모바일 오버레이 외에는 사용하지 않는다.

## 타이포그래피

- 기본 글꼴: Pretendard, Apple SD Gothic Neo, system-ui
- 페이지 제목: 34px / 700 / 1.25
- 섹션 제목: 20px / 700 / 1.35
- 본문: 15px / 400 / 1.55
- 컨트롤: 14px / 600 / 1.3
- 보조 정보: 12px / 500 / 1.4

## 허용된 첫 화면 문구

- Quilo
- 2학년 4반
- 오늘, 일정, 시간표, 구성원, 알림 기록, 설정
- 오늘의 학급 운영
- 일정 추가, 시간표 수정
- 다가오는 수행평가, 다음 알림

## 구성 요소

- `AppShell`: 내비게이션과 작업 영역, 편집 패널 소유
- `Sidebar`: 현재 메뉴, 가입 현황, 관리자 프로필
- `PageHeader`: 학급 선택과 주요 작업
- `TimetableTable`: 교시별 수업 행과 편집 동작
- `EventList`: 수행평가 마감 상태와 상세 진입
- `NotificationList`: 예정/성공/실패 상태
- `EventDrawer`: 일정 생성·편집 양식
- `BottomNavigation`: 760px 이하 모바일 전용

## 상호작용

- `일정 추가`는 우측 편집 패널을 연다.
- 일정 저장 후 목록과 예정 알림을 즉시 갱신한다.
- 시간표는 요일별로 일괄 저장하며 빈 교시는 허용한다.
- 실패한 알림은 원인과 재시도 가능 여부를 표시한다.
- 모든 버튼은 키보드 포커스 링과 44px 이상의 터치 영역을 가진다.

## 의도적 차이 허용

- 시안의 교사 이름은 실제 MVP 데이터 모델에 필수값이 아니므로 선택 필드로 둔다.
- 화면 폭이 좁으면 우측 패널을 항상 표시하지 않고 필요할 때 여는 시트로 바꾼다.
- 실제 날짜와 데이터는 서버 응답을 사용하므로 시안의 샘플 날짜에 고정하지 않는다.
