# 아키텍처 개요

Quilo의 공개 아키텍처는 아래와 같은 고수준 흐름으로 이해할 수 있습니다.

```mermaid
flowchart LR
  A["사용자 자료"] --> B["입력 검증"]
  B --> C["자료 구조화"]
  C --> D["AI 기반 초안 생성"]
  D --> E["데이터 일관성 확인"]
  E --> F["문서 렌더링"]
  F --> G["사용자 검토 및 다운로드"]
```

## 설계 원칙

- **원본 우선:** 사용자가 제공한 자료와 측정 데이터가 생성 문장보다 우선합니다.
- **검증 가능한 출력:** 확인할 수 없는 값이나 근거를 임의로 만들지 않는 것을 목표로 합니다.
- **문서 완결성:** 화면의 답변이 아니라 다시 열고 편집할 수 있는 문서를 결과물로 다룹니다.
- **개인정보 최소화:** 서비스 운영에 필요하지 않은 입력 내용을 장기 보관하지 않는 방향으로 설계합니다.
- **실패의 명시:** 파싱이나 문서 생성이 실패하면 성공한 결과처럼 숨기지 않습니다.

## 공개 수준

이 문서는 제품 개념을 설명하기 위한 것입니다. 데이터 모델, 요청 형식, 프롬프트, 보정 규칙, 렌더러 내부 구조, 배포 절차와 운영 방어 로직은 공개하지 않습니다.
