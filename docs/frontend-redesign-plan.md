# Quilo 전체 프론트엔드 리디자인 실행 기준

이 문서는 2026-07-11 시작한 전체 프론트엔드 리디자인의 범위와 안전 경계를 고정한다. 목표는 학생 전용 도구처럼 보이는 좁은 인상을 줄이고, 학습·연구·창작·개발을 연결하는 신뢰도 높은 AI 작업 플랫폼으로 확장하는 것이다.

## 제품 방향

- 현재 핵심 고객: 고등학생과 대학생을 포함한 학습, 연구, 문서 작업 사용자
- 확장 정체성: 학습, 연구, 문서, 번역, 창작, 코딩을 연결하는 AI 작업 플랫폼
- 핵심 메시지: `Quilo | 학습과 연구를 위한 AI Workspace`
- 시각 방향: 넓은 여백, 강한 타이포그래피, 절제된 코발트 블루, 실제 제품 화면 중심의 설명
- 언어 원칙: 주요 설명은 한국어, 제품명·섹션 라벨·짧은 핵심 문장에만 영어를 제한적으로 사용

## 페이지 그룹

1. 마케팅: 홈, 가이드, 예시, 패치노트, 요금, 약관·개인정보·환불
2. 핵심 작업: 보고서 생성, 공부, 수행평가, 물리 스튜디오, 창작 스튜디오, 에디터, 파일 채팅, 번역
3. 도구: 도구 인덱스와 PDF·이미지·그래프·회귀·글자수·수식 변환기
4. 개발자: `/developers.html`, API·CLI·SDK·Codex/ChatGPT 연결
5. 앱: Quilo Desktop과 Live Translator의 독립 소개·다운로드 페이지
6. 계정: 로그인, 회원가입, 이메일 인증, 프로필과 보안 설정
7. 운영: 관리자 전체 화면

## 12차 실행 순서

1. 전체 읽기 전용 감사와 데이터 변경 금지선 확정
2. 공통 디자인 토큰·상단바·내비게이션·모바일 셸 구축
3. 홈페이지와 중앙 Quilo Bot 개편
4. 일반 사용자 페이지 전체 개편
5. 실제 작업 환경을 3열 AI 워크스페이스로 개편
6. 개발자 페이지 자연스러운 편입
7. 앱별 독립 소개·다운로드 페이지 구축
8. 로그인·회원가입·인증 흐름 개편
9. 관리자 페이지 전체 개편
10. 반응형·접근성·성능·SEO 정리
11. 회귀 검증과 사용자 데이터 무변경 확인
12. 실제 작업 장면 기반 제품 광고 애니메이션 제작

## 현재 상태 (2026-07-12)

- 1~8차: 구현 완료. 홈·공통 셸·작업 폼·개발자·앱·인증 화면을 새 디자인으로 연결했다.
- 9차: 관리자 화면을 Quilo Operations 대시보드로 개편했다. 기존 DOM/API 계약을 보존했다.
- 10차: 1280px·933px·390px 셸 회귀, 가로 넘침, 접근성 상태, 제거 기능의 404를 자동 점검한다. 모바일의 별도 시각 전면 개편은 사용자 요청대로 보류한다.
- 11차: 로컬 fixture와 읽기 전용 브라우저 검증을 완료한다. 운영 데이터 쓰기 요청은 허용하지 않는다.
- Render: 환경변수·헬스체크·롤백 기준은 `docs/render-release-checklist.md`에 정리한다. 실제 운영 배포는 별도 승인 후 진행한다.
- 12차: 사용자 요청으로 이번 작업 범위에서 제외한다.

중간 확인은 2차, 5차, 8차, 11차 완료 시점에 한다. 8차 확인 뒤에는 사용자 의견을 받은 후 9차를 시작한다.

## 데이터 안전 경계

- 운영 사용자, 크레딧, 권한, 보고서, 파일, API 토큰, 사용량 데이터를 생성·수정·삭제하지 않는다.
- 관리자 `POST`/`PATCH`/`DELETE` API를 호출하지 않는다.
- 사용자 프로필, 비밀번호, 파일 삭제, API 키 생성·삭제 API를 자동 테스트하지 않는다.
- 실제 생성, 결제, 이메일 발송, 토큰 발급, 크레딧 차감 요청을 자동 테스트하지 않는다.
- 데이터베이스 스키마나 마이그레이션을 변경하지 않는다.
- QA는 정적 렌더링, 읽기 전용 라우트, 로컬 fixture와 브라우저 상호작용 검증으로 제한한다.
- 기존 폼의 `name`, `id`, 제출 URL과 서버 계약은 해당 기능 차수에서 명시적으로 검증하며 보존한다.

## 공통 셸 원칙

- 모든 공개 페이지는 동일한 Quilo 브랜드, 제품 분류, 개발자·앱·리소스 진입 구조를 공유한다.
- 작업 화면은 마케팅 내비게이션을 복제하지 않고 밀도 높은 앱 셸 변형을 사용한다.
- 관리자 화면은 같은 토큰을 쓰되 운영 작업의 정보 밀도와 위험 동작 구분을 우선한다.
- `prefers-reduced-motion`, 키보드 탐색, 충분한 대비, 44px 이상의 터치 영역을 기본값으로 둔다.
- 공통 셸은 읽기 전용이며 사용자 데이터 API를 호출하지 않는다.
