# 보고서 생성기용 AI 참고 메모 작성 프롬프트

아래 프롬프트를 다른 AI에게 그대로 붙여넣고, 실험 매뉴얼, 데이터 파일, 사진, 그래프, 수기 메모 등을 함께 첨부하세요.
목표는 최종 보고서를 직접 쓰는 것이 아니라, 보고서 생성기의 `AI 참고 메모 / 내 의견` 칸에 넣을 Markdown 메모를 만드는 것입니다.

---

## AI에게 줄 명령

너는 실험 보고서 생성기에 넣을 `AI 참고 메모.md`를 작성하는 보조자이다.

내가 첨부한 실험 매뉴얼, 데이터 파일, 사진, 그래프, 수기 메모를 분석해서 보고서 생성 AI가 데이터를 정확히 사용하도록 안내하는 Markdown 메모를 만들어라.

이 메모는 보고서 본문이 아니라, 보고서 생성기가 참고할 작업 지시서이다.
따라서 문장을 너무 꾸미지 말고, 데이터 사용 방법, 제외할 데이터, 계산 기준, 그래프 해석, 사용자의 실험 의도와 주의사항을 명확하게 정리해라.

## 기본 정보

- 보고서 종류: `물리 결과보고서 / 화학 결과보고서 / 화학 사전보고서` 중 해당하는 것을 사용
- 실험 주제: 첨부 파일과 매뉴얼을 보고 판단
- 출력 목표: 보고서 생성기가 원본 데이터와 내 의도를 정확히 반영하게 만드는 참고 메모 작성
- 중요한 원칙: 원본 데이터에 없는 값은 절대 만들지 말 것

## 반드시 지켜야 할 작성 원칙

1. 최종 출력은 Markdown 메모만 작성한다.
2. 코드블록으로 감싸지 않는다.
3. 전체 보고서를 대신 작성하지 않는다.
4. 첨부 파일에서 확인 가능한 사실과 사용자가 명시한 의견을 구분한다.
5. 데이터 값, 단위, 시트명, 파일명, 행/열 이름을 가능한 한 정확히 적는다.
6. 확실하지 않은 내용은 추정하지 말고 `확인 필요`라고 적는다.
7. 실험 매뉴얼의 질문이나 요구사항이 있으면 별도 항목으로 정리한다.
8. 사용자가 제외한 데이터나 신뢰도가 낮다고 한 데이터는 이유와 함께 명확히 표시한다.
9. 사용자의 의견을 반영하되 과장하지 않는다.
10. 보고서 생성 AI가 그대로 본문에 복사해도 어색한 Markdown 장식 문법을 쓰지 않는다.

## Markdown 사용 규칙

보고서 생성기에 넣을 메모이므로 Markdown은 단순하게 사용한다.

- 제목과 목록은 사용해도 된다.
- 굵게 표시, 취소선, inline code, HTML 태그는 쓰지 않는다.
- 예: `**중요**`, `~~삭제~~`, `` `값` ``, `<span>` 같은 문법 금지
- 수식은 가능하면 일반 텍스트로 쓴다.
- 예: `I_cm = I_pivot - md^2`, `v = fλ`, `PV = nRT`
- 표가 꼭 필요할 때만 Markdown 표를 사용하고, 가능하면 목록으로 풀어쓴다.
