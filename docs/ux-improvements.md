# Quilo UX·사용자 편의 개선 리포트 (2026-06-26)

8개 관점(핵심 플로우·폼·피드백·온보딩·카피·모바일·베타·신뢰)으로 멀티에이전트 감사 → 80건 발견을 중복 제거·우선순위화한 결과. 이미 처리된 토큰·다크모드·기계적 접근성은 제외.

## 🎯 우선순위 개선 (impact high/medium, effort M+) — 가성비 순

1. **확인 모달 + 완료 화면의 크레딧 투명성 일원화** — 모달에 '차감 크레딧·차감 후 잔액'을 1순위 행으로, 완료 화면에 '이번 생성 N크레딧 차감·남은 N' 배지. 학생은 크레딧으로 결제하는데 모달은 $/₩만, 완료 화면은 차감액 미표시(모달의 '완료 후 표시' 약속도 불이행). `impact:high / effort:S~M`
2. **클라이언트 크레딧 사전 점검 + 부족 시 회복 동선** — 모델/모달 단계에서 보유<필요면 인라인 경고, 생성 버튼 '크레딧 부족' 비활성화, '더 싼/무료 모델로' 1클릭·문의 링크. 지금은 다 채우고 누른 뒤 접힌 로그에 평문 한 줄로만. `impact:high / effort:M`
3. **생성 오류 시 원클릭 '다시 시도' + 행동 중심 메시지** — 직전 FormData 보관 후 모달 없이 재전송, 실패 로그 자동 펼침, '크레딧 차감 안 됨·입력 유지' 명시. 스트림 끊김은 일시 원인 많아 가치 큼. `impact:high / effort:M`
4. **출력 설정(형식·모델) 발견 가능하게 + 현재 선택 요약** — 핵심 가치인 한글(.hwpx)과 비용을 좌우하는 모델이 '선택 설정' 아코디언에 숨음. 밖으로 빼거나 summary에 현재 값 표시. `impact:high / effort:M`
5. **선택 기억(reportType·모델·형식·글꼴) + '기본값' 카드 정합** — 마지막 성공 생성값 localStorage 복원(또는 '지난번처럼' 칩). 출력 형식 기본값도 폼마다 엇갈림. `impact:medium / effort:M`
6. **긴 textarea 자동 저장(draft) + 이탈 경고** — 자유 보고서 작성 지시(8000자)·메모(12000자)가 오류·뒤로가기·새로고침에 통째로 소실. 디바운스 localStorage 저장 + 복구 + beforeunload. `impact:high / effort:M`
7. **필수 입력 체크리스트를 실제 입력 상태와 연동** — '체크리스트'가 정적 텍스트(죽은 UI). 폼 상태 바인딩해 ✓/미충족 강조 + 섹션 스크롤. `impact:medium / effort:M`
8. **물리 결과 폼에 학번 인라인 입력/표시** — 학번 필수인데 입력칸 없어 끝까지 채우고 alert로 설정 탭 튕김. 표지 미리보기 + 그 자리 저장. `impact:high / effort:M`
9. **진행 화면에 경과 타이머 + ETA 유지** — 2~8분 작업인데 경과·잔여 사라져 멈춘 줄 알고 이탈. 모달의 estimate를 진행 화면으로 넘겨 '예상 X분·경과 0:23' + 단계 한 줄. `impact:medium / effort:M`
10. **모델 선택 마이크로카피 통일 + 용도 안내** — 'Opus 4.8 · 정밀 · 4크레딧'/'GPT mini · 빠름 · 무료' 형식 통일, '정확도면 Opus, 초안이면 mini' 한 줄, 기본에 '추천' 배지. `impact:medium / effort:S~M`
11. **모바일 단계 '다음/이전' 버튼 + 진행 영역 고정** — 섹션 채우면 상단 스텝 탭이 화면 밖; 진행 영역도 스크롤 시 사라짐; 노치 단말 FAB가 버튼 가림. full-width '다음→', sticky 진행 헤더, dvh/safe-area. `impact:high / effort:M`
12. **가이드·예시 진입점 + 베타 기능 반영** — 가이드가 보고서 3종에만 머물러 메뉴와 어긋남. 히어로 보조 링크 + 종류 선택 후 해당 가이드 앵커, guide/examples에 자유·수행평가·스튜디오·번역 추가. `impact:high~medium / effort:S~M`

## 한 줄 총평

가장 크게 개선될 1~2가지: **(1) 비용 투명성** — 학생이 실제 쓰는 단위인 '크레딧'을 확인 모달·완료 화면·잔액 경고에서 일관되게 보여주고 부족 시 회복 동선 제공. **(2) 막다른 길 제거** — 인증·학번·크레딧·필수 파일 같은 차단 조건을 폼을 다 채우기 *전에* 알려주고, 실패 시 원클릭 재시도와 입력 보존으로 2~8분 대기가 헛수고가 되지 않게.
