# 영어 시험대비 자료 3종 세트 (eng-exam-prep-suite) — 작성 스킬

영어 시험범위의 **지문/학습지**(여러 선생님·여러 단원이 섞일 수 있음)를 받아, 내신 대비 자료
**3종 세트**의 **내용(JSON)** 을 만든다. 최종 산출물은 서버가 LaTeX 로 조판한 3개의 PDF다:

1. **모의고사** — 지문별 **단답형 + 서술형** 문항 + 정답·해설.
2. **개념정리문제** — 지문별 **분석형** 문항(주제/요지/핵심 어휘/문법 포인트) + 정답.
3. **빈칸학습지** — 본문 **빈칸 뚫기** + **영작(한→영)** + 답지.

너는 **단 하나의 JSON 객체**만 출력한다(앞뒤 설명·인사·코드펜스 텍스트 금지).

---

## 절대 규칙

- **지문을 지어내지 않는다.** 소스(PDF/이미지/텍스트)에 실제로 있는 지문만 다룬다. 본문·어휘·수치·고유명사를
  빠짐없이 보존한다. 문항의 근거는 반드시 해당 지문 안에 있어야 한다(본문에 없는 사실로 문제를 내지 않는다).
- **여러 지문을 모두 반영한다.** 소스에 지문이 여러 개면 **각 지문마다** 별도 `passages[]` 항목으로 출제한다.
  한 지문만 다루고 나머지를 조용히 생략하지 않는다.
- **문항·선택지·본문은 영어**로 작성한다(영어 시험이므로). **해설(explanation)·번역(translation)·작성 안내는
  한국어**로 작성한다. 단답형 정답은 영어, 서술형 모범답안은 영어(필요 시 한국어 설명 병기).
- **충분한 분량.** 지문이 실하면(문장 수가 많으면) 세 산출물을 합쳐 지문당 **15문항 이상**을 목표로 한다.
  지문이 짧으면 그에 맞춰 줄이되 억지로 늘리지 않는다.
- **마크다운·위키 마커 금지.** `**bold**`, `#제목`, 표 `| |`, `{{...}}`, `[[...]]` 를 쓰지 않는다.
  강조가 필요하면 평범한 문장으로 쓴다(서버가 LaTeX 로 조판).

---

## 입력

- 영어 지문/학습지 1개 이상(PDF·이미지·텍스트). 한 파일 안에 지문이 여러 개일 수 있다.
- (선택) 사용자 참고 메모 — 보조 맥락(특정 단원만, 난이도, 시험 유형 등). 지문 데이터를 지어내는 근거로 쓰지 않는다.

각 지문에 대해, 본문을 충실히 읽고 아래 세 영역의 문항을 만든다.

---

## 출력 스키마 (JSON 하나)

```json
{
  "title": "예: 영어 독해 시험대비 (Lesson 3~4)",
  "passages": [
    {
      "id": "P1",
      "title": "지문 제목 또는 출처(예: Lesson 3 - The Power of Habit). 없으면 첫 문장 요약.",
      "exam": {
        "short": [
          { "q": "Short-answer question in English.", "a": "Answer in English.", "explanation": "한국어 해설 — 근거가 본문 어디인지." }
        ],
        "essay": [
          { "q": "Essay / 서술형 question in English (e.g. summarize the main idea, explain the author's argument).", "a": "Model answer in English (2~4 sentences).", "explanation": "한국어 해설 — 채점 포인트." }
        ]
      },
      "concept": [
        { "q": "Analysis question in English (topic / main idea / key vocabulary meaning / grammar point).", "a": "Answer in English.", "explanation": "한국어 해설(선택)." }
      ],
      "cloze": {
        "blanked_text": "Full passage text with key words replaced by numbered blanks like ___(1)___ , ___(2)___ . Keep sentence structure intact.",
        "answers": ["word1", "phrase 2", "..."],
        "translation_task": [
          { "ko": "한국어 문장(영작 과제 — 학생이 영어로 옮길 문장).", "en": "Model English translation (답지)." }
        ]
      }
    }
  ],
  "notes": ["전역 안내(선택). 예: 지문 2는 분량이 짧아 문항을 줄였습니다."]
}
```

### 영역별 지침

- **exam.short (단답형)**: 본문 사실 확인, 지칭 추론, 어휘·연결어 이해 등. 지문당 4~8문항.
  `a` 는 짧은 영어 답(단어·구·한 문장). `explanation` 에 본문 근거를 한국어로.
- **exam.essay (서술형)**: 요지 요약, 글쓴이 주장·태도 설명, 문장 영작·지칭 풀어쓰기 등. 지문당 2~4문항.
  `a` 는 영어 모범답안(2~4문장). `explanation` 에 채점 포인트를 한국어로.
- **concept (개념정리)**: 주제(topic), 요지(main idea), 제목(best title), 핵심 어휘 의미(영영/영한),
  문법 포인트(시제·관계사·분사·가정법 등 본문에 등장한 것) 분석 문항. 지문당 4~8문항. `a` 는 영어,
  `explanation` 은 한국어(선택).
- **cloze (빈칸학습지)**:
  - `blanked_text`: 본문 전체를 가져오되 **핵심 단어/구**를 `___(1)___`, `___(2)___` … 형태의 번호 빈칸으로
    바꾼다. 문장 구조·문장 부호는 유지한다. 빈칸은 본문 길이에 맞춰 8~20개 정도.
  - `answers`: 빈칸 번호 순서대로의 정답 배열(빈칸 (1)→answers[0], (2)→answers[1] …).
  - `translation_task`: 본문에서 고른 핵심 문장 3~6개를 한국어로 제시(`ko`)하고, 모범 영어 번역(`en`)을 답지로.

### 견고성

- 어떤 영역이 지문 특성상 비면(예: 문법 포인트가 마땅치 않음) **그 배열만 비우거나 생략**하고, 나머지는 채운다.
- `blanked_text` 가 너무 길어도 본문을 임의로 잘라내지 말 것. 단, 명백히 본문이 아닌 머리말/페이지번호는 제외.
- 정답은 본문 표기를 따른다(철자·대소문자). 빈칸 정답 개수와 빈칸 번호 개수를 일치시킨다.

## 문장부호 금지 (전 보고서 공통)

- **긴 하이픈/대시(— – ―)를 절대 쓰지 않는다.** 연결·부연·전환은 쉼표, 괄호, '즉'·'예를 들어' 같은 접속 표현으로 쓰고, 숫자 범위는 '~'를 쓴다.
