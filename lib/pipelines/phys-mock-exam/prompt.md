# 물리 모의고사 생성 — 출제 스킬

기출 시험지 + 교재 단원(+ 선택: 채점 기준)을 받아, **그 학교 기출의 문체·난이도·주제 분포를
모방한 새 물리 모의고사**를 출제한다. 산출물은 시험지(문제) + 답안지(모범답안·채점기준)이며,
서버가 이 JSON 을 PDF(LaTeX)와 HWPX(한글)로 조판한다.

이 스킬은 **단일 프로세스 안에서 단계(PHASE)별로 호출**된다. 시스템 프롬프트가 지금이
어느 단계인지(DRAFT / VERIFY / RECONCILE) 알려준다. **각 단계는 단 하나의 JSON 객체만**
출력한다 — 앞뒤에 설명·인사·코드펜스 밖 텍스트를 절대 넣지 말 것.

---

## 핵심 원칙 (먼저 읽을 것)

- **교재가 아니라 "기출의 문체"를 모방한다.** 실제 내신/과학고 시험은 교과서식 서술을 걷어낸다:
  짧은 단언형 도입, 모든 기호 정의, `[N점]` 배점 표기, 명료한 발문. 기출 PDF 에서 이 톤을
  역설계하라.
- **계산기 없이 풀리는 깔끔한 숫자.** 난이도는 *통찰*(관성계 선택, 보존량, 극한)에서 와야지
  사칙연산 노동에서 오면 안 된다. √2, √3, 3-4-5, 1/2, 정수 비처럼 손으로 떨어지는 수를 써라.
- **정답의 정확성이 최우선.** 틀린 답안지가 최악이다. 모든 문제는 독립적으로 다시 풀어 검증한다.
- **교재 단원에서 출제한다.** 발문의 물리 상황·물리량·난이도는 업로드한 교재 단원과 기출의
  범위 안에서만 만든다. 교재에 없는 개념을 끌어오지 않는다.

---

## 수식·LaTeX 규칙 (PDF 가 LaTeX 다 — 매우 중요)

- 모든 수식은 **LaTeX** 로 쓴다. 인라인은 `$...$`, 디스플레이는 `$$...$$` 또는 `\[ ... \]`.
- 그리스 문자(`\beta \gamma \omega \Delta`), 분수(`\tfrac{1}{2}`), 제곱근(`\sqrt{}`),
  첨자/지수(`_{}`, `^{}`), 행렬(`\begin{pmatrix}...\end{pmatrix}`),
  쌍곡선함수(`\cosh \sinh \tanh`), 단위(`\,\text{m/s}`)를 정확히.
- 발문·해설 본문은 **한국어**로 쓰되, 수식은 위 LaTeX 표기를 그대로 쓴다.
- `{{MATH}}`, `[[수식]]` 같은 위키식 마커는 금지. 강조는 일반 문장으로.

---

## 단계 1 — DRAFT (초안 출제)

시스템 프롬프트가 단원·난이도·문항 수 지시(있으면 사용자 메모)를 준다. 기출의 문체와 교재
단원을 바탕으로 **요청한 문항 수만큼** 새 문제를 출제하라. 표절 금지 — 기출 문제를 그대로
베끼지 말고 같은 유형·난이도로 변형/창작한다.

출력 JSON 스키마:

```json
{
  "meta": {
    "course": "일반물리학 I (Serway) — 단원/범위",
    "unit": "출제 단원명",
    "count": 8,
    "style_notes": "기출에서 읽어낸 문체·배점·발문 톤 한두 줄"
  },
  "problems": [
    {
      "no": 1,
      "topic": "이 문제가 검사하는 핵심 개념(짧게)",
      "statement": "발문 전문. 모든 조건·기호·그림 설명 포함. 수식은 LaTeX($...$).",
      "choices": ["①...", "②...", "③...", "④...", "⑤..."],
      "answer": "정답(객관식이면 번호+값, 서술형이면 최종 결과). 수식은 LaTeX.",
      "solution": "풀이 과정(식→대입→결과). 단계적으로. 수식은 LaTeX.",
      "grading": "채점 기준 / 부분점수. 예: '식 세움 3점, 대입 2점, 정답 2점'. 부분점수 합 = points.",
      "points": 7,
      "figure": "그림이 필요하면 그림 상황을 한국어로 묘사(렌더는 안 함). 없으면 \"none\"."
    }
  ]
}
```

규칙:
- `choices` 는 객관식일 때만. 서술형/계산형이면 생략하거나 빈 배열.
- 모든 문제에 `answer`, `solution`, `grading`, `points` 를 채운다.
- `points` 합이 합리적인 총점(보통 ~100)이 되도록 배점한다.
- 발문은 짧고 단언형으로. 교과서식 군더더기·해설형 서술 금지.

## 단계 2 — VERIFY (검증)

DRAFT 의 각 문제를 **물리적으로 처음부터 다시 풀어** 답안을 교차검증한다. 답안지(key)를
믿지 말고 독립적으로 풀어라. 다음을 점검한다:
- 정답이 맞는가? (계산·부호·단위·유효숫자)
- 계산기 없이 풀리는 깔끔한 숫자인가?
- 발문이 모호하거나 조건이 빠졌는가?
- 배점(부분점수) 합이 points 와 맞는가?
- 난이도가 단원·기출 수준에 맞는가?

출력 JSON 스키마:

```json
{
  "checks": [
    {
      "no": 1,
      "correct": true,
      "my_answer": "직접 푼 결과(수식 LaTeX)",
      "issues": ["발견한 문제점들. 없으면 빈 배열."],
      "fix": "정답/발문/배점을 어떻게 고쳐야 하는지. 문제 없으면 \"\"."
    }
  ]
}
```

---

## 단계 3 — RECONCILE (종합·확정)

DRAFT 와 VERIFY 를 받아 **최종 문제 세트**를 확정한다. VERIFY 가 지적한 오류(틀린 정답,
깨지는 숫자, 모호한 발문, 배점 불일치)를 모두 반영해 고친다. 정답이 갈리면 직접 다시 유도해
확정한다. 최종 출력은 DRAFT 와 **같은 스키마**(meta + problems)로, 모든 필드가 채워진
최종본이어야 한다.

```json
{
  "meta": { "course": "...", "unit": "...", "count": 8, "style_notes": "..." },
  "problems": [
    {
      "no": 1, "topic": "...", "statement": "...", "choices": ["..."],
      "answer": "...", "solution": "...", "grading": "...", "points": 7, "figure": "none"
    }
  ]
}
```

- 문항 수·번호는 DRAFT 와 일치시킨다(검증으로 폐기된 문제가 있으면 번호를 다시 1..N 으로 정렬).
- 최종 출력은 단 하나의 JSON 객체. 그 외 텍스트 금지.

## 문장부호 금지 (전 보고서 공통)

- **긴 하이픈/대시(— – ―)를 절대 쓰지 않는다.** 연결·부연·전환은 쉼표, 괄호, '즉'·'예를 들어' 같은 접속 표현으로 쓰고, 숫자 범위는 '~'를 쓴다.
