1. AI 참고 메모 작성 가이드

보고서 폼의 AI 참고 메모 / 내 의견 칸(또는 .md/.txt 첨부)은 생성 AI가 내 데이터와 의도를 정확히 반영하도록 돕는 작업 지시서입니다. 보고서 본문이 아니므로 짧게 한두 줄만 써도 되고, 길게 쓸 거면 아래 양식대로 .md 파일로 만들어 함께 올리면 됩니다.

  • 메모는 데이터를 대체하지 않습니다. 원본 데이터와 충돌하면 데이터가 우선입니다.
  • 제외한 데이터는 이유를 꼭 적으세요. 이유가 없으면 보고서가 임의로 빼거나 지어내지 않습니다.
  • 같은 사실은 한 번만 명확히 — 보고서에는 최대 2회 정도만 반영됩니다.
  • 장식 문법(굵게, 인라인 코드, HTML 태그)은 본문에 새어 나갈 수 있으니 피하세요. 제목·목록은 OK.
  • 메모는 학교 양식·작성 기준을 덮어쓰지 못합니다. “양식 무시하고~”는 적용되지 않습니다.

# 실험 참고 메모

## 실험 기본 정보
- 보고서 종류: 물리 결과보고서
- 실험 주제: 역학적 에너지 보존 — 경사면 위 카트의 운동
- 첨부한 주요 파일:
  - run_data.cap (PASCO Capstone 원자료, 포토게이트 2개 + 모션 센서)
  - clean.xlsx (내가 직접 정리한 측정값, 시행별 속도와 높이)
  - photo_track.jpg (실험 장치 사진)
  - graph_v2_vs_h.png (속도 제곱 - 높이 그래프 스크린샷)

## 데이터 사용 우선순위
1. clean.xlsx — 내가 정리하면서 단위와 이상치를 한 번 걸러낸 값이라 이걸 우선으로 써 줘.
2. run_data.cap — clean.xlsx에 없는 시계열·센서 세부값이 필요할 때만 참고.
3. graph_v2_vs_h.png — 추세선 형태와 기울기 확인용. 픽셀로 값을 추정하지는 말 것.

## 사용할 데이터 / 계산 기준
- 사용할 시트: clean.xlsx 의 "summary" 시트.
- 사용할 열: 높이 h (m), 속도 v (m/s), 시행 번호.
- 단위: 길이 m, 속도 m/s, 에너지 J. 질량 m = 0.512 kg (메모 기준값으로 사용).
- 계산: 각 시행에서 mgh 와 (1/2)mv^2 를 구해 비교하고, 보존율(%) = (운동E / 위치E) x 100 으로 정리.
- 평균은 시행 2~5만 사용 (아래 제외 항목 참고). 표준편차도 같은 범위로.
- v^2 = 2 g h 가 이론식이므로, v^2 - h 그래프의 기울기를 2g 와 비교해 줘.

## 제외하거나 조심할 데이터
- 제외한 것: 1차 시행 (시행 1).
- 이유: 출발 직후 포토게이트가 살짝 흔들려 첫 속도값이 비정상적으로 작게 찍혔음.
  평균·표준편차 계산에서 빼 줘. 단, 실험 전체가 실패한 것은 아니니 오차 요인에서만 짧게 언급.
- 트랙 이음새 구간(높이 0.35 m 부근)에서 카트가 미세하게 튀는 느낌이 있었음 — 마찰·충격 손실의 근거로만 활용.

## 사진·그래프
- photo_track.jpg: 경사면 + 카트 + 포토게이트 2개가 보이는 장치 전경. 캡션 "실험 장치 구성".
- graph_v2_vs_h.png: v^2 (세로) - h (가로) 그래프. 거의 직선이며 기울기가 2g 근처. 원점 부근은 약간 아래로 처짐.

## 매뉴얼 질문 / 토의 항목
- 측정한 보존율이 100%보다 작게 나온 주된 원인은 무엇인가?
- 마찰을 줄이면 v^2 - h 그래프의 기울기는 어떻게 변하는가?
- 공기 저항과 트랙 마찰 중 어느 쪽 영향이 더 컸다고 볼 수 있는가?

## 내 의견 / 강조하고 싶은 점
- 위치에너지가 운동에너지로 바뀌는 경향은 잘 나타났고, 보존율이 대체로 90% 이상이라는 점을 강조하고 싶음.
- 손실분은 트랙 마찰과 포토게이트 정렬 오차로 설명하는 흐름으로 결론을 써 줘.

## 하지 말아야 할 것
- clean.xlsx 에 없는 측정값을 새로 만들지 말 것.
- 제외한 1차 시행을 평균에 다시 넣지 말 것.
- 그래프 스크린샷에서 보이지 않는 수치를 단정하지 말 것.
- 내 메모를 과장해서 "에너지가 완벽히 보존됐다" 또는 "실험이 실패했다"는 식으로 몰지 말 것.

자료를 넣으면 보고서가 이렇게 나옵니다. 아래 실제 DOCX 예시에서 문서 구조와 출력 품질을 확인할 수 있습니다.

화학 사전보고서

실험 매뉴얼 PDF → 사전보고서 (DOCX·HWPX)
  • 매뉴얼의 이론·원리 키워드를 펼쳐 쓴 이론적 배경
  • 시약 물성·실험 기구·실험 과정을 학교 양식으로 정리
  • 그림이 필요한 자리에 placeholder + 사진 설명

화학 결과보고서

데이터(엑셀·CSV) · 사진 → 결과 분량 (DOCX·HWPX)
  • 평균·표준편차·백분율 오차 자동 계산
  • 표·차트(그래프)와 실험 사진 삽입
  • 사전보고서 뒤에 이어지는 결과·분석·결론

물리 결과보고서

.cap · 엑셀·CSV · 사진 → 결과보고서 (DOCX·HWPX)
  • 실험 파트별 표·그래프·정량 분석
  • 이론값 비교와 경향성 해석
  • 결과 요약·오차 분석이 담긴 결론
